In this tutorial, we will use Suki to build a RESTful API. This will allow us to have an api that will get all tasks, create a task, and complete and delete a task. It will return all this information in JSON format.

1. Install Suki

Suki is bundled with an executable, aptly named suki(1). If you install suki globally with npm you'll have it available from anywhere on your machine:

$ npm install -g suki

2. Create a application

A good beginning is half done. Suki can create a template application for you:

$ suki init task

Where "task" is the name of your application.

After execute the init command, Suki will create a "task" directory in the current path.

3. Define the model

A typical Suki application is consisting of two parts. One is controllers, and the other is models. Controllers is related to the router, which we will see later in this tutorial. Models is where we defined our

$ suki create-model task

4. Define the controller

$ suki create-controller task
module.exports = class extends require('suki').Controller

  # POST /tasks
  create: ->
    task = @db.Task.create { title: @req.body.title }
    @res.json task

  # GET /tasks
  index: ->
    @res.json @db.Task.findAll()

  # GET /tasks/:id
  get: ->
    @res.json @_task

  # DELETE /tasks/:id
  destroy: ->
    @_task.destroy()
    @res.json { "message": "deleted" }
